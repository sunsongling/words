# 修仙合成放置微信小游戏 项目文档（个人极简可落地版）

## 一、项目概述

### 1\.1 项目背景

微信小游戏生态中，**合成\+放置\+修仙**是个人开发者最容易上线、流量稳定、审核通过率极高的品类。这类游戏玩法轻量、碎片化、无需操作难度、用户留存高、停留时长高。

市面上大部分修仙合成游戏功能臃肿、广告密集、配置复杂。本项目主打**极简修仙、轻松合成、离线挂机、境界升级**，纯休闲玩法，零上手门槛，非常适合微信用户碎片化游玩。

本项目为**个人独立开发版本**，架构极简、零服务器、低成本、可快速上线、可长期迭代。

### 1\.2 项目定位

- 类型：休闲放置 \+ 合成升级 \+ 修仙养成

- 受众：全年龄段休闲玩家，摸鱼、挂机、轻度游戏用户

- 特点：无需肝、无需氪、离线收益、合成突破、境界提升

- 开发定位：个人零成本、低代码、易维护、易上线

### 1\.3 设计原则（个人开发核心）

- **玩法极简**：只保留合成、挂机、境界、存档四大核心，无复杂系统

- **零后端**：使用微信云开发存储存档，无需服务器

- **零运维**：全自动存档、全自动离线收益

- **高过审率**：无竞技、无排名、无充值、无敏感内容，纯单机休闲

## 二、玩法与需求分析

### 2\.1 核心玩法（V1\.0 必做）

核心闭环：**挂机产出灵气 → 合成高阶灵物 → 提升修为 → 突破境界 → 解锁更多高阶灵物**

- 场景内自动刷新低阶灵物

- 两个相同等级灵物拖动合成更高一级

- 每级灵物持续产出修为/灵气

- 修为累积达到阈值自动突破境界

- 离线挂机可累积离线修为收益

- 自动存档、上线自动结算离线收益

### 2\.2 游戏内容设计

#### 2\.2\.1 灵物合成体系（1\-10阶）

从低到高：灵草 → 灵木 → 灵石 → 灵玉 → 灵丹 → 法器 → 宝器 → 灵宝 → 仙器 → 神器

规则：**两个同阶合成一个高一阶**，所有高阶灵物持续产出更高修为。

#### 2\.2\.2 修仙境界体系

基础十境（适配休闲玩法）：

炼气 → 筑基 → 金丹 → 元婴 → 化神 → 炼虚 → 合体 → 大乘 → 渡劫 → 飞升

每突破一个境界，全局灵气产出倍率提升，解锁更高阶灵物刷新概率。

#### 2\.2\.3 离线挂机系统

- 退出游戏后自动计时

- 上线自动计算离线时长收益

- 最大离线收益上限（防止数值爆炸）

#### 2\.2\.4 自动存档系统

游戏数据实时云端存档，换设备、卸载重装不会丢档。

### 2\.3 V1\.0 功能清单（可直接上线）

- 灵物拖拽合成

- 场景自动刷新低级灵物

- 修为数值实时增长

## 四、游戏核心逻辑设计

### 4\.1 合成逻辑

- 场景内随机生成1阶灵物

- 监听触摸拖拽，两个同等级碰撞合并

- 合并成功生成高1阶灵物

- 每个灵物实时产生对应修为值

### 4\.2 修为与境界逻辑

- 所有场上灵物每秒产出修为

- 修为累计达到境界阈值自动突破

- 突破后全局产出倍率提升

- 境界越高，刷新高阶灵物概率越高

### 4\.3 离线收益逻辑

- 退出游戏记录时间戳

- 下次进入计算离线时长

- 根据当前产出倍率结算离线修为

- 设置最大离线8小时收益，防止数值崩坏

### 4\.4 存档逻辑

- 游戏数据实时缓存本地

- 场景变化、境界变化、合成变化自动云端存档

- 进入游戏自动读取云端存档

### 13\.3 核心游戏逻辑 game\.js

```javascript
const app = getApp()
const db = wx.cloud.database()
// 全局游戏数据
let gameData = {
  repair: 0,       // 当前修为
  level: 0,        // 当前境界
  rate: 1.0,       // 产出倍率
  itemList: [],    // 场上灵物
  lastTime: 0      // 最后离线时间
}
// 最大灵物数量、最大离线时长
const MAX_ITEM = 30
const MAX_OFFLINE_TIME = 28800

Page({
  data: {
    levelName: "",
    repair: 0,
    rate: 1.0,
    progress: 0
  },
  onShow() {
    this.loadSave()
  },
  onHide() {
    gameData.lastTime = new Date().getTime()
    this.saveData()
  },
  // 读取云端存档
  async loadSave() {
    let res = await db.collection("game_user").where({openid:app.globalData.openid}).get()
    if(res.data.length > 0){
      gameData = res.data[0]
      // 结算离线收益
      this.calcOffline()
    }else{
      // 新用户初始化数据
      gameData = {
        repair:0,level:0,rate:1.0,itemList:[],lastTime:new Date().getTime()
      }
    }
    this.updateUI()
    this.startGameLoop()
    this.startRefreshItem()
  },
  // 计算离线收益
  calcOffline() {
    let now = new Date().getTime()
    let diffTime = parseInt((now - gameData.lastTime) / 1000)
    if(diffTime < 10) return
    diffTime = Math.min(diffTime, MAX_OFFLINE_TIME)
    // 计算总产出
    let totalPower = 0
    gameData.itemList.forEach(item=>{
      totalPower += app.globalData.itemPower[item.level] * gameData.rate
    })
    let offlineRepair = parseInt(totalPower * diffTime)
    gameData.repair += offlineRepair
    // 检测境界突破
    this.checkLevelUp()
  },
  // 检测境界突破
  checkLevelUp() {
    let limit = app.globalData.levelLimit
    while(gameData.level < 9 && gameData.repair >= limit[gameData.level+1]){
      gameData.level++
      gameData.rate = app.globalData.rateList[gameData.level]
    }
  },
  // 游戏主循环：修为增长
  startGameLoop() {
    setInterval(()=>{
      let add = 0
      gameData.itemList.forEach(item=>{
        add += app.globalData.itemPower[item.level] * gameData.rate
      })
      gameData.repair += parseInt(add)
      this.checkLevelUp()
      this.updateUI()
    },1000)
  },
  // 自动刷新1阶灵物
  startRefreshItem() {
    setInterval(()=>{
      if(gameData.itemList.length < MAX_ITEM){
        gameData.itemList.push({
          level:0,
          x: Math.random()*250,
          y: Math.random()*300
        })
        this.updateUI()
      }
    },3000)
  },
  // 合成逻辑（同等级合并）
  mergeItem(index1,index2) {
    let item1 = gameData.itemList[index1]
    let item2 = gameData.itemList[index2]
    if(item1.level === item2.level && item1.level <9){
      // 移除两个旧灵物，生成高阶灵物
      gameData.itemList.splice(index2,1)
      gameData.itemList.splice(index1,1)
      gameData.itemList.push({
        level: item1.level+1,
        x: item1.x,
        y: item1.y
      })
      this.updateUI()
      this.saveData()
    }
  },
  // 更新界面数据
  updateUI() {
    let nextLimit = app.globalData.levelLimit[gameData.level+1] || app.globalData.levelLimit[9]
    let progress = gameData.level >=9 ? 100 : parseInt(gameData.repair / nextLimit * 100)
    this.setData({
      repair: gameData.repair,
      levelName: app.globalData.levelName[gameData.level],
      rate: gameData.rate,
      progress
    })
  },
  // 自动存档
  saveData() {
    db.collection("game_user").where({openid:app.globalData.openid}).update({
      data: {...gameData, openid: app.globalData.openid}
    })
  }
})
```
