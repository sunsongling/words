# 宠物疫苗提醒微信小程序项目文档（个人开发版）

## 一、项目概述

### 1\.1 项目背景

随着宠物饲养普及率持续提升，猫狗疫苗接种、驱虫、体检是宠物健康养护的核心刚需。大部分宠物主人为上班族、学生群体，日常工作生活繁忙，极易忘记宠物疫苗到期时间、年度体检、内外驱虫等养护节点。

目前市场上同类工具多为综合宠物平台的附属功能，广告多、功能冗余、操作复杂，暂无轻量化、无广告、专注**疫苗与养护提醒**的极简工具类小程序。基于个人轻量化开发需求，打造一款功能单一、操作简单、精准提醒的宠物疫苗提醒小程序，满足普通养宠用户的核心刚需。

### 1\.2 项目定位

轻量化、免费、无广告、纯工具类微信小程序，**专注宠物疫苗、驱虫、体检到期提醒**，无需复杂注册，上手即用，适配个人独立开发，代码简洁、部署简单、维护成本极低。

### 1\.3 设计原则（个人开发核心）

- **极简功能**：舍弃冗余社交、商城、资讯功能，只保留记录\+提醒\+查询核心功能

- **低开发成本**：采用微信小程序原生框架，无需后端服务器，依托微信云开发实现数据存储

- **零维护压力**：无用户审核、无内容运营、无付费体系，纯工具自动化运行

- **易用性优先**：页面简洁，操作步骤少，老人、新手均可快速上手

## 二、需求分析

### 2\.1 用户需求

- 支持添加多只宠物信息（猫狗通用）

- 可记录每只宠物的疫苗、驱虫、体检记录（含接种时间、疫苗类型、下次到期时间）

- 自动计算下次养护时间，临近到期自动提醒

- 可查看历史所有养护记录，支持修改、删除记录

- 内置常见宠物疫苗周期参考，新手无需自行查询

#### 2\.2\.2 进阶迭代功能（V1\.1及后续版本）

## 四、核心功能详细设计

### 4\.1 首页模块（核心主页）

页面展示：用户所有已添加宠物卡片、到期提醒汇总、快捷添加记录入口

核心逻辑：自动筛选**即将到期（7天内）、已过期**的养护项目，高亮提醒，优先展示风险项，用户打开小程序即可直观查看待办事项。

### 4\.2 宠物信息管理模块

功能：添加、编辑、删除、查看宠物信息，支持多宠物管理

可录入字段：宠物昵称、宠物类型（猫/狗）、性别、年龄、头像、备注信息

设计亮点：删除宠物同步删除该宠物所有养护记录，避免数据冗余，无需手动清理。

### 4\.3 养护记录管理模块（核心功能）

支持三类核心记录：疫苗接种、体内外驱虫、健康体检

每条记录可录入字段：养护类型、具体项目（狂犬疫苗、猫三联、犬四联等）、执行时间、下次到期时间、备注、凭证图片（可选）

自动计算逻辑：内置通用养护周期规则，用户选择养护项目、填写执行时间后，系统**自动生成下次到期时间**，无需手动计算。

通用周期规则（内置默认，可自定义修改）：

- 猫狗基础疫苗：首年3针，次年1针，之后每3年1针

- 狂犬疫苗：每年1针

- 体内驱虫：每月1次

- 体外驱虫：每月1次

- 常规体检：每年1次

### 4\.4 消息提醒模块（核心亮点）

采用微信官方**一次性订阅消息**（个人小程序无需企业资质，免费开通）

提醒规则：

- 到期前7天、到期当天、过期3天分别推送提醒

- 用户首次添加记录时，授权订阅消息，后续自动推送，无需重复授权

- 所有提醒消息直接推送至微信服务通知，打开即可查看详情

### 4\.5 记录查询与统计模块

- 按宠物筛选记录：单独查看某一只宠物的所有养护数据

- 按类型筛选：单独查看疫苗/驱虫/体检记录

- 时间轴展示：记录按时间倒序排列，清晰查看养护历史

### 4\.6 帮助中心模块

内置新手科普内容：猫狗疫苗接种时间表、驱虫注意事项、常见问题解答，无需联网查询，提升小程序实用性。

## 六、核心功能逻辑流程

### 6\.1 新增养护记录流程

用户选择宠物 → 选择养护类型 → 选择具体项目 → 填写完成时间 → 系统自动计算下次到期时间 → 保存至云数据库 → 授权订阅提醒 → 流程完成。

### 6\.2 定时提醒推送流程

云函数每日定时触发 → 遍历用户所有未过期记录 → 判断是否达到提醒时间 → 调用微信订阅消息接口 → 向用户推送提醒通知。

### 6\.3 数据安全逻辑

通过 openid 隔离用户数据，每个用户仅可查询、修改、删除自己的宠物数据，无法查看他人信息，保障用户隐私。

## 九、项目风险与开发注意事项

### 9\.1 核心开发注意事项

- **用户数据隔离**：所有数据必须绑定用户openid，严格区分不同用户数据，杜绝数据串访问题，保障用户隐私安全。

- **云函数定时规则**：微信云开发定时任务仅支持每日单次触发，需配置固定时段（推荐凌晨2点）批量检测到期数据，避免频繁触发消耗额度。

- **订阅消息适配**：个人小程序仅支持一次性订阅消息，需在用户新增养护记录时主动引导授权，未授权用户无法推送提醒，前端需做好弹窗提示。

- **数据容错处理**：针对网络异常、重复提交、空数据场景做容错判断，避免页面报错、数据重复存储。

- **图片存储规范**：宠物头像、凭证图片统一上传云存储，禁止本地缓存，避免换设备数据丢失。

### 9\.2 项目风险与解决方案

- **风险1：云开发免费额度不足**：解决方案：个人用户日均访问量低，免费额度完全够用，可在云开发后台配置用量预警，超量可升级低价套餐。

- **风险2：消息推送失效**：解决方案：增加前端手动刷新提醒状态逻辑，同时保留本地时间计算，即使云函数异常，用户打开小程序仍可查看到期提醒。

- **风险3：数据误删丢失**：解决方案：新增删除二次确认弹窗，同时依托云开发数据回溯功能，可恢复近期误删数据。
